Enhanced Answer: How does a computer boot?

Boot Sequence: Power On ‚Üí BIOS/UEFI POST ‚Üí Bootloader ‚Üí OS Kernel ‚Üí User Space

  1. Power-On Self Test (POST): BIOS/UEFI checks hardware (CPU, RAM, peripherals)
  2. Bootloader: Loads from boot device (GRUB for Linux, Windows Boot Manager for Windows)
  3. Kernel Loading: OS kernel loaded into memory, initializes drivers
  4. Init/System Services: System services start (Windows: services.msc, Linux: systemd/init)
  5. Login Prompt: User authentication presented
Why This Wins
Shows complete chain understanding from hardware to OS‚Äîcritical for investigating boot-sector malware, rootkits, UEFI threats. Demonstrates systems thinking beyond "click power button."
Enhanced Answer: Where would malware persist on Windows?

Common Persistence Locations:

  • HKLM\Software\Microsoft\Windows\CurrentVersion\Run
  • HKCU\Software\Microsoft\Windows\CurrentVersion\Run
  • C:\Users\[User]\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup
  • Scheduled Tasks: C:\Windows\System32\Tasks\
  • Services: HKLM\SYSTEM\CurrentControlSet\Services
  • DLL Hijacking: Placing malicious DLLs in system path
  • WMI Event Subscriptions: Fileless persistence technique
Why This Wins
Demonstrates practical IR knowledge. Specific registry paths + fileless techniques (WMI) shows you're beyond basics. This is exactly what Tier 1/2 SOC analysts investigate daily.
